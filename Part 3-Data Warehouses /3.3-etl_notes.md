## ETL Decisions

### Decision 1 — Date Format Standardization
Problem: Raw data had mixed formats (DD/MM/YYYY strings, DD-MM-YYYY, and Excel serial numbers like 45272 and 45144).  
Resolution: Converted every date to standard DATE 'YYYY-MM-DD' (used Python datetime for serials + manual parsing for strings) before loading into dim_date.

### Decision 2 — Category Casing Normalization
Problem: Inconsistent casing ("electronics" vs "Electronics", "Grocery" vs "Groceries").  
Resolution: Standardized all categories to Title Case ("Electronics", "Clothing", "Groceries") in dim_product so grouping in Q1 works correctly.

### Decision 3 — Handling Missing Store City
Problem: Several rows had empty/NULL store_city (e.g., TXN5033 for Mumbai Central, TXN5044, etc.).  
Resolution: Replaced empty values with "Unknown" in dim_store to avoid NULLs and enable clean reporting without losing rows.
